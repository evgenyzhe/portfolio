# Project: Digital Product Designer Portfolio

## Goal
Build a polished portfolio website for a senior digital product designer.
The site should feel serious, corporate, premium, and memorable for recruiters.

## Stack
- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- Responsive layout: mobile, tablet, desktop
- Static content where possible

## Design direction
- Clean corporate visual style
- Strong typography
- Spacious layout
- Case-study focused
- Recruiter-friendly structure
- No playful startup vibe
- Avoid generic portfolio clichés
- Use subtle motion only when it improves clarity

## Site structure
Required pages/sections:
1. Hero section
2. About / professional positioning
3. Key strengths
4. Selected case studies
5. Process / how I work
6. Experience highlights
7. Contact / CTA

## Content priorities
The visitor should quickly understand:
- I am a senior digital product designer
- I work with complex digital products, financial scenarios, analytics, dashboards, and B2B/B2C interfaces
- I can translate business logic into clear user experience
- I can work from research and hypotheses to delivery and post-release validation
- I balance business needs, user needs, and technical constraints

## UX rules
- First screen must immediately explain who I am and what value I bring
- Case cards should show problem, role, result, and product area
- Every section should help hiring managers evaluate me
- Avoid decorative blocks without meaning
- Navigation should be simple and predictable

## Code rules
- Use reusable components
- Keep files clean and readable
- Avoid unnecessary dependencies
- Use semantic HTML
- Maintain accessibility basics: headings, contrast, alt text, keyboard-friendly links/buttons
- Optimize for performance
- Do not hardcode repeated styles if a component is better
- All static assets (images, icons, previews) must use `getAssetPath()`. Never use raw `/images/...` paths.

## Output rules
Before making changes:
1. Briefly explain the intended changes.
2. Then implement.
3. After implementation, summarize changed files and how to run/check the project.

## Do not
- Do not create a childish or overly colorful portfolio
- Do not use lorem ipsum unless explicitly requested
- Do not invent fake metrics unless marked as placeholder
- Do not add backend unless needed
